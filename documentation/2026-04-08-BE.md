## Client ↔ API

- Vite **`server.proxy`** for **`/api`** → `http://localhost:3000`; optional **`VITE_API_URL`**; **`VITE_USE_MOCK_API=true`** forces fixtures.
- **`apiJson`**, **`ApiError`**, **`apiUrl`** in `packages/client/src/api/client.ts`.
- **`/login`**, JWT in **`sessionStorage`**, **`AuthContext`**, **`RequireAuth`** around owner shell; header sign-out.
- **`mockApi.ts`** delegates to **`fetchPublicMenu`**, **`fetchOwner*`**, etc., unless mocks are on; **`clearReadCaches`** on login/logout.
- Guest **`readPublicMenu`** → **`GET /api/v1/public/menus/:menuId`**. Owner reads for menus, categories, menu hub (composed **`venueName` / `menuName`**), category page, item editor.
- Item editor URL: **`/menus/:menuId/items/:itemId/edit`**.

## Server

- **`OwnerItemsPort.listItems`** + **`GET /api/v1/owner/menus/:menuId/items`** with optional **`?categoryPublicId=`** (owner category table, all visibility states).
- **`MenuItemDto.visibleOnMenu`** on owner list payloads where applicable.
- Adjustments for **MongoDB 6** `findOneAndUpdate` return shape, typed **`$pull`**, JWT **`expiresIn`** typing (as needed for clean **`tsc`**).

## UX / errors

- **`SnackbarProvider`** + **`errorNotifier`**: failed **`apiJson`** calls show **`GuestFilterSnackbar`** with server **`error.message`** (or a network hint). **`loginRequest`** passes **`showErrorSnack: false`** so the login form’s inline error is not duplicated.
